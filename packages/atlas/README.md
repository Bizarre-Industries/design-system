# @bizarre/atlas

Deterministic vector renderer for the governed Astronomical Atlas extension. This is a private working package while the owner-selected left Continuous Lens Aperture v2 remains governed-provisional; neither the aperture nor its dependent outputs are publishable assets.

The renderer implements one synthetic, single-mass field algorithm. It does not consume a seed, model multiple masses, reproduce an observed sky, or claim physical-production evidence. Every SVG embeds the algorithm and model versions, the exact source-configuration hash, representation, governed aperture version and path hash, approved orientation, trajectory state, optical size, appearance, and production profile.

## API

```js
import { renderAtlas, renderAperture } from '@bizarre/atlas';

const field = renderAtlas({
  orientation: 18,
  representation: 'contour',
  appearance: 'dark',
  opticalSize: 'medium',
  productionProfile: 'screen'
});

const aperture = renderAperture();
```

Allowed field orientations are `-38`, `-18`, `18`, and `38` degrees. Representations are `contour`, `spectral`, `bands`, `dots`, and `hatch`. Micro output deliberately omits field lines rather than shrinking medium-detail geometry below policy.

The Continuous Lens working master is the owner-selected left direction reconstructed as four tangent-continuous cubic quadrants at a fixed `-14deg` rotation. The nominal unrotated construction bounds are `480 x 220` (`24:11`); its separate `1.618` construction-ratio token remains a provisional calibration value and is not the silhouette aspect ratio. The outline has no straight segment, corner, chamfer, wedge, notch, key cut, or polygonal join.

Generated derivatives live below `generated/`; `generated/config.json` makes the renderer configuration portable, while `generated/manifest.json` owns every generated byte hash and concise SVG provenance. `brand/provisional-assets.json` supplies their nonpublishable governance state. These files are available for controlled design reconstruction and review, not public package consumption. Only the evidence-free `screen` working profile is accepted; print, vinyl, and one-color production require separate governed transformations and validation before they can become renderer profiles.
